# The Dragon's Touch v0.10.0-dev — Simple View Foundation Contract

## Purpose

v0.10 begins the human-readable UI presentation layer.

v0.9 proved the Replacement Candidate Engine can produce stable engine-side data and report surfaces. v0.10 should not rewrite the engine first. It should make the already-verified information easier for normal users, younger users, and testers to understand inside the UI.

## v0.10 goal

Create a Simple View that answers:

- What does my deck need?
- What cards from my collection look useful?
- What full-card-pool examples might help later?
- Why is this card being shown?
- Is this card owned?
- Is this card safe for my budget/table/bracket?
- Is this an automatic swap?

## Core Simple View rule

Simple View must never imply the app made a final deck-building decision for the user.

Use language like:

- Suggested to review
- Possible fit
- Worth considering
- Manual review
- Example only
- Not automatically swapped

Avoid language like:

- Must add
- Must cut
- Guaranteed upgrade
- Best card
- Automatically replace

## Required Simple View sections

### 1. Deck Need Summary

Human-readable summary of what the deck appears to need.

Display fields:

- Main need
- Why it matters
- Confidence
- Caution, if any

### 2. Collection Candidates

Cards from the user's collection that are plausible fits.

Display fields:

- Card name
- Owned / collection status
- Role
- Why this card?
- Why this deck?
- Confidence
- Caution
- Manual review reason, if any

### 3. Full-Pool Examples

Full-card-pool examples that may help explain possible options.

Display fields:

- Card name
- Not confirmed owned / example only
- Role
- Why it might help
- Budget unknown
- Bracket/table-fit caution
- Not automatically swapped

### 4. Manual Review Cards

Cards that should not be treated as direct recommendations yet.

Display fields:

- Card name
- Reason for manual review
- What to check
- Confidence or caution

### 5. Protected / Do-Not-Cut Notes

Cards or card types the engine should avoid casually targeting.

Display fields:

- Card name or category
- Protection reason
- User preference source, if available

## Labels Simple View must support

- Owned
- Not confirmed owned
- Collection-first
- Full-pool example
- Manual review
- Budget not checked
- Bracket/table-fit caution
- Color identity not verified
- Low confidence
- Not automatically swapped
- Combo examples suppressed unless enabled

## v0.10 non-goals

- Do not add automatic swaps.
- Do not add live pricing yet.
- Do not add live API calls.
- Do not make full-pool examples override collection-first.
- Do not turn Simple View into a full recommendation ranking engine.
- Do not remove the advanced/dev report.
- Do not redesign the whole UI before creating the Simple View contract.

## Recommended v0.10 first UI step

Add a Simple View placeholder/panel that can eventually display:

- Need Summary
- Collection Candidates
- Full-Pool Examples
- Manual Review
- Safety Notes

The first version may be static/placeholder-driven as long as the labels and section structure are correct.
