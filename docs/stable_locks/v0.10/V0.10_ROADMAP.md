# The Dragon's Touch — v0.10 Roadmap

## v0.10 theme

**Simple View / Human-Readable UI Presentation Layer**

v0.10 should translate the v0.9 engine/report surfaces into a clearer UI experience.

## v0.10.0-dev — Simple View Foundation Plan + UI Contract

Status: current step.

Goal:

- Define what Simple View should show.
- Define required labels.
- Define what Simple View must not imply.
- Add a verifier that checks the contract exists.

## v0.10.1-dev — Simple View UI Placeholder

Goal:

- Add a Simple View area/panel to the existing UI.
- Do not wire full parsing yet.
- Show placeholder sections:
  - Deck Need Summary
  - Collection Candidates
  - Full-Pool Examples
  - Manual Review
  - Safety Notes

Pass criteria:

- UI opens.
- Existing run flow still works.
- Placeholder Simple View is visible or reachable.
- No backend behavior changes.

## v0.10.2-dev — Simple View Report Loader Hook

Goal:

- Read the generated normal report file.
- Identify report sections needed for Simple View.
- Do not deeply parse every card yet.
- Show raw extracted section snippets in the Simple View panel.

Pass criteria:

- Report file loads.
- Simple View can show extracted sections.
- Existing advanced report remains available.
- No crashes when sections are missing.

## v0.10.3-dev — Simple View Candidate Card Extraction MVP

Goal:

- Extract candidate card rows from:
  - Replacement Candidate Engine Preview
  - Full-Card-Pool Fallback Preview
  - Exact Full-Pool Candidate Preview
  - Need-Aligned Exact Preview Addendum

Pass criteria:

- Candidate cards appear as simple UI cards.
- Each card has role/source/caution where available.
- Full-pool examples are not labeled owned.

## v0.10.4-dev — Simple View Safety Labels

Goal:

- Add visible badges/labels:
  - Owned
  - Not confirmed owned
  - Full-pool example
  - Budget not checked
  - Bracket/table-fit caution
  - Manual review
  - Not automatically swapped

Pass criteria:

- Safety labels are visible.
- Full-pool examples remain clearly separated from owned candidates.
- Automatic swaps remain No.

## v0.10.5-dev — Simple View Tester Readability Pass

Goal:

- Make wording easier for testers and casual players.

- Plain-language candidate explanations
- Clearer warnings
- Less AI/dev wording

Pass criteria:

- A tester can understand what is being suggested without reading the full markdown report.
