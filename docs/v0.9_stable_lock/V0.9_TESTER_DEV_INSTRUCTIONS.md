# The Dragon's Touch — v0.9 Tester / Dev Instructions

## Purpose

v0.9 is a stable engine-side checkpoint for replacement candidate logic. Testers should focus on whether the tool gives useful, bounded, understandable replacement candidate information without pretending to make automatic choices for the player.

## Basic test flow

1. Run a normal batch or single-deck analysis.
2. Check that the report generates successfully.
3. Confirm the report includes:
   - Replacement Candidate Engine Preview
   - Full-Card-Pool Fallback Preview
   - Collection-first boundary
   - Automatic swaps: No
4. Look for whether replacement candidates feel relevant to:
   - The commander
   - The deck's stated plan
   - The needed role
   - Collection-first preference
   - Budget/bracket caution
5. Check whether any full-pool examples are clearly labeled as not automatically owned or not automatic swaps.

## Combo-aware test flow

Use PowerShell:

```powershell
$env:MTG_COMBO_AWARENESS_ENABLED = "1"
$env:MTG_COMBO_AWARENESS_ARTIFACT = "both"
py main.py
```

Then verify that Combo Awareness content does not break:

- Replacement Candidate Engine Preview
- Full-Card-Pool Fallback Preview
- Exact preview safety labels
- Aggregate output generation

## Final known-good verifier

```powershell
py tools\verify_v0.9.8_stable_candidate_final_regression.py Outputs\v0.9.5.5.2 Outputs\v0.9.5.6_combo_on
```

Expected final verdict:

```text
PASS — v0.9.8 stable-candidate final regression passed.
```

## What feedback is most useful

- Candidate feels off-plan
- Candidate seems too generic
- Candidate should have been collection-first but was not
- Candidate looks like it is being treated as owned when it is not
- Report section is confusing
- Combo-aware content changes replacement candidate behavior unexpectedly
- Aggregate output fails or is missing

## What feedback is less useful at v0.9

- UI is not pretty enough
- Report is not fully kid-friendly
- Report is too technical

Those are valid concerns, but they belong mostly to v0.10 Simple View / UI work.
