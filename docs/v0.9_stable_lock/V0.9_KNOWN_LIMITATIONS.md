# The Dragon's Touch — v0.9 Known Limitations

## Scope limitations

v0.9 locks the replacement-candidate engine foundation. It does not claim to be the final human-facing experience.

## Known limitations

- The report is still AI/dev-friendly, not fully layperson-friendly.
- The future human-readable layer belongs mainly in v0.10 Simple View / UI.
- Exact full-pool examples are exploratory examples, not a complete ranking engine.
- Full-card-pool fallback does not replace collection-first authority.
- Budget is labeled but not live-price checked.
- Bracket/table-fit is cautioned but not fully evaluated as a live social contract.
- Full-pool examples are not claimed as owned.
- Automatic swaps remain disabled.
- Combo examples remain suppressed unless combo optimization is explicitly enabled.
- Archetype regression coverage is keyword-based and broad, not a true semantic archetype classifier.
- No live API calls are required for the v0.9 lock.
- No automatic deck edits are made.

## What v0.9 does safely

- Keeps collection-first recommendations primary.
- Labels full-pool fallback examples honestly.
- Preserves source/ownership boundaries.
- Preserves confidence/caution surfaces.
- Preserves no-automatic-swap boundaries.
- Preserves combo-aware report compatibility.
- Preserves batch aggregate output generation.

## What v0.10 should improve

- Human-readable Simple View.
- Cleaner UI display of why a candidate is shown.
- Better visual separation between collection-owned candidates and full-pool examples.
- Clearer labels for manual review, caution, and do-not-recommend reasons.
- Easier tester-facing summary of replacement needs.
