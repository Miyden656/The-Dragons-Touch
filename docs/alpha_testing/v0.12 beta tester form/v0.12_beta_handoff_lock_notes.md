## What Testers Should Do

1. Extract the ZIP.
2. Open `README_START_HERE.txt`.
3. Install requirements:

   ```powershell
   py -m pip install -r requirements.txt
   ```

4. Launch the app:

   ```powershell
   py desktop_ui_launcher.py
   ```

5. If needed, use fallback launch:

   ```powershell
   py ui\dragons_touch_pyside6_workstation.py
   ```

6. Open **Settings → Data Setup**.
7. Run:
   - Download / Update Scryfall
   - Download / Update Combo Data
   - Build Combo Index
8. Select a deck.
9. Complete Review Setup.
10. Run analysis.
11. Open the report in Report Viewer.
12. Fill out `docs/beta_tester_feedback_form_v0.12.4.md` if anything is confusing or broken.

## Known Limitations

- This is a source-run beta, not an installer.
- There is no EXE in the handoff package.
- Users need Python installed.
- Data setup requires internet access when downloading Scryfall and Commander Spellbook data.
- Huge generated data files are intentionally not bundled.
- Combo Awareness depends on local data setup and a built combo index.
- The Report Viewer is still intended as a practical beta viewer, not a full polished markdown renderer.
- New deck-building engine features are intentionally out of scope for v0.12.

## What Not To Expect

v0.12 does not include:

- EXE packaging
- installer support
- code signing
- Commander Discovery
- Replacement Candidate Engine expansion
- Combo Awareness expansion
- live pricing
- Archidekt login
- automatic deck swaps
- automatic data downloads during analysis
- major backend rewrite

## Files Intentionally Excluded From Handoff

- `.exe` files
- `.bat` files unless intentionally approved later
- `dist/`
- `build/`
- `build_specs/`
- `_internal/`
- `MockUP/` files or folders
- old alpha ZIPs
- obsolete feedback clutter
- huge raw Scryfall JSON files
- huge combo JSON files
- combo index files when users are expected to build them locally
- personal/internal notes
- `__pycache__/`
- `.pyc` files
