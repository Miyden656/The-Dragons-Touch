The old `docs` folder was reduced from a historical working folder into a clean replacement folder focused on active beta handoff, project reference, stable lock notes, and compact patch history.

- Active v0.12 source-run beta support files at docs root.

- Project executive summary, philosophy integration notes, parity/refactor references, and terminal test command notes.
- Stable lock references for v0.8, v0.9, and v0.10.

## Removed from active docs and archived separately

- `Combo Match tester/` bulk stress tests and reconciliation reports.
- `dev documentation note/` long patch-by-patch dev scratch history.
- `Patch notes/` full historical patch stream, except recent v0.12/v0.13 readmes.
- Old alpha feedback tracking folders and empty tester folders.
- Duplicate copies of beta forms already preserved in root or `beta_testing/`.

## Why

This keeps the replacement `docs` folder useful for running, testing, and explaining the project without dragging the entire development-history workspace into the clean handoff package.
